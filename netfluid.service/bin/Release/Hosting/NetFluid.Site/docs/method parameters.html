To set parameters of your exposed methods all you need to do is to define them as normal parameters.
Each type of route defines where to get parameters:
  • Route: Parameters are taken from POST and GET data.If the same variable is defined in both, POST is prevailing
  • ParametrizedRoute: Parameters are taken from parts of the Uri according to their order in the method
  • RegexRoute: Parameters are taken from regex groups in the Uri according to their name
C# primitive types and enums are automatically parsed, for complex types you need to define a static method called Parse that take as string as input and return your complex type (as example: a function which take an id as input and return the corresponding object from the database)

Simple route with data taken from POST

This simple method take his parameters from POST data, if a parameter is missing C# default value is used (0 for numbers, null for string, lowest value for enums) 
[Route("/add")]
public object Add(string name, string surname, string email, string tel)
{
	if (name != null && surname != null && email != null && tel != null)
	{
		Person.Save(new Person
		{
			Name = name,
			Email = email,
			Id = Security.UID(),
			Surname = surname,
			Telephone = tel
		});
	}

	return new FluidTemplate("./UI/index.html");
}

Taking parameters from the Uri

To do that the code will be exactly the same, just change the Route attribute into ParametrizedRoute.
You can define both attribute on the same method to unlock all the possibilities 
[ParametrizedRoute("/add")]
public object Add(string name, string surname, string email, string tel)
{
	//same code of above
}
In this case to invoke our method will all the parameters we have to visit

http://mywebapp.com/add/Matteo/Fabbri/matteo@phascode.org/00393939148691

Taking parameters from regex name groups

Note: Named groups use a C# specific notation (?<GroupName>RegexRule) 

namespace _9.RegexRoutes
{
    class Program : FluidPage
    {
        //Ex: http://localhost:8080/Mattew
        [RegexRoute("/(?<name>[a-zA-Z]+)")]
        public string Person(string name)
        {
            return "This is a person, his name is " + name; 
        }

        //Ex: http://localhost:8080/232131
        [RegexRoute("/(?<number>[\\d]+)")]
        public string Person(int number)
        {
            return "This is a number, his value is " + number;
        }

        static void Main(string[] args)
        {
            Engine.Interfaces.AddLoopBack(8080);
            Engine.Start();
            Console.ReadLine();
        }
    }
}

Parsing complex types

To parse a complex type, independently by the type of the route, add a Parse method to your object. In this example we got a routed method taking an array of Person and delete them all from the database. 
[Route("/delete")]
public object Delete(Person[] person)
{
	if (person != null)
		foreach (var p in person)
			Person.Delete(p);

	return new FluidTemplate("./UI/index.html");
}
To transform our array of id into an array of person all we need to do is to add the Parse method to our Person class: 
public class Person
{
	public string Email;
	public string Id;
	public string Name;
	public string Surname;
	public string Telephone;

	public static Person Parse(string id)
	{
		Person p;
		Database.Get(id, out p);
		return p;
	}

}
