NetFluid web applications can be compiled as libraries or as self-hosted application.
Each webapp can contain unlimited virtual hosts running in the same C# AppDomain or you can set FastCGI forwarding on specific host-names.
If no virtual hosts are specified, the current context is served by the default web application.

The current assembly is always loaded as default web application

  • Interfaces : Accepts sockets, create contexts and handles threads of requests
  • Engine : Manage all necessary assemblies and hosts.
  • Fowarder : Forwards specific connections to others FastCGI endpoint.
  • Default web application : Serves each request unsatisfied by the other hosts into the current application
  • Virtual host applications : Netfluid assemblies loaded under specific hostnames
