To set interfaces, public folders, sessions managements, sockets timeout and all the NetFluid parameters you can do it programmatically or from App.config file

To load app configuration you can use Engine.LoadAppConfiguration which load the default app configuration or Engine.LoadAppConfiguration(sting:confpath) wich load another file

Engine parameters

  • int Timeout: Set sockets timeout in milliseconds.On timeout client socket is closed and resources released. If missing the default value is 300 ms.
  • string LogPath: Set the path used by engine logger If missing the default value is "./AppLog.txt"
  • bool DevMode: If true show on console the context flow. Remember to turn it off in production. Turned on, it show exception on client response and real status codes, otherwise in case of error the response status code is "Bad Request" and no messages is shown. If missing the default value is false
  • int MaxPostSize: Maximum client payload in bytes, if it exceeds the context is closed as "Bad Request". If missing the default value is 268435456 (256MB)
  • int SessionDuration: Session duration in seconds, expired all the session data of current user are trashed. Sessions are started and renewed automatically if used. If missing the default value is 3600 seconds (1 hour)

Interface parameters

HTTP interfaces and HTTPS interfaces are declared in the same way, if they got a PFX certificate they are HTTPS otherwise they are HTTP

  • string IP: IP on which listen. Required
  • int Port: IP on which listen. If missing the default value is 8080
  • string Certificate: PXF certificate path for HTTPS listening. If missing the interface will be HTTP

Public folders

Map a physical path to an Uri.Files and directories inside a public folder are freely explorable and downloadable by clients. A Public folder can be immutable which means that all files inside are kept in memory by NetFluid engine without taking care of changing, for high performances, ideal for static contents of your web application like styles and JavaScript.
Public folders can also be mapped on specific virtual host to setup in place contents system.

  • string RealPath: Physical path of the public folder Required
  • string UriPath: Uri to map Required
  • bool Immutable: If true, files are memory cached and all changes ignored
  • string Host: Map the folder on a virtual host

Example configuration

<?xml version="1.0" encoding="utf-8" ?>
<configuration>
  <configSections>
    <section name="NetFluidSettings" type="NetFluid.Settings, NetFluid"/>
    <!-- ATTENTION DUE TO A C# BUG configSections MUST BE THE FIRST CHILD OF CONFIGURATION -->
  </configSections>
  <startup> 
        <supportedRuntime version="v4.0" sku=".NETFramework,Version=v4.5" />
  </startup>
  <!-- NetFluid settings will all parameters -->
  <NetFluidSettings Timeout="500" LogPath="./AppLog.txt" LogLevel="Exception" DevMode="false" MaxPostSize="268435456" SessionDuration="3600">
    <Interfaces>
      <!-- HTTP INTERFACE ON PORT 8080 -->
      <Interface IP="127.0.0.1"/>
      
      <!-- HTTP INTERFACE ON PORT 80 -->
      <Interface IP="127.0.0.1" Port="80"/>

      <!-- HTTPS INTERFACE ON PORT 444 -->
      <Interface IP="127.0.0.1" Port="444" Certificate="my.pfx"/>
    </Interfaces>
    <PublicFolders>
      
      <!-- PUBLIC FOLDER -->
      <PublicFolder RealPath="./upload" UriPath="/upload"/>
      
      <!-- MEMORY CACHED PUBLIC FOLDER -->
      <PublicFolder RealPath="./Public" UriPath="/" Immutable="true"/>

      <!-- PUBLIC FOLDER EXPOSED AS VIRTUALHOST -->
      <PublicFolder RealPath="./supb" UriPath="/" Host="true"/>
      
    </PublicFolders>
  </NetFluidSettings>
</configuration>
